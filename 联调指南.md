双击运行：`E:\Codes\call-go\start.bat`

### 方法 2：手动启动

打开新的命令行窗口，执行：

```bash
# 刷新环境变量（如果需要）
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")

# 进入项目目录
cd E:\Codes\call-go

# 启动服务
go run main.go
```

**验证后端是否启动成功：**

在浏览器访问：`http://localhost:8080/health`

应该看到：
```json
{"status":"ok"}
```

## 第三步：启动前端服务

打开新的命令行窗口，执行：

```bash

cd E:\Codes\call

# 安装依赖（如果还没安装）
pnpm install

# 启动 H5 开发服务
pnpm run dev:h5
```

前端将在 `http://localhost:3000` 启动（具体端口看控制台输出）

## 前后端接口对应关系

### 游戏管理

| 功能 | 前端调用 | 后端接口 |
|------|---------|---------|
| 获取游戏列表 | `loadGames()` | `GET /api/v1/games` |
| 获取我的游戏 | `loadMyGames()` | `GET /api/v1/games/my` |
| 创建游戏 | `createGame()` | `POST /api/v1/games` |
| 加入游戏 | `joinGame()` | `POST /api/v1/games/join` |
| 退出游戏 | `leaveGame()` | `POST /api/v1/games/:id/leave` |

### 存取分管理

| 功能 | 前端调用 | 后端接口 |
|------|---------|---------|
| 存分 | `deposit()` | `POST /api/v1/transactions/deposit` |
| 取分 | `withdraw()` | `POST /api/v1/transactions/withdraw` |
| 获取余额 | `loadUserBalance()` | `GET /api/v1/transactions/balance/:game_id` |
| 获取交易记录 | `loadGameTransactions()` | `GET /api/v1/transactions/game/:game_id` |
| 获取参与者列表 | `loadGameParticipantBalances()` | `GET /api/v1/transactions/participants/:game_id` |

## 测试流程

### 1. 测试后端 API

使用 Postman 或 curl 测试：

```bash
# 健康检查
curl http://localhost:8080/health

# 获取游戏列表
curl http://localhost:8080/api/v1/games

# 创建游戏
curl -X POST http://localhost:8080/api/v1/games \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"测试游戏\",\"description\":\"这是一个测试游戏\"}"
```

### 2. 测试前后端联调

1. 确保后端已启动（http://localhost:8080）
2. 确保前端已启动（http://localhost:3000）
3. 在浏览器打开前端页面
4. 尝试创建游戏、加入游戏、存取分等操作
5. 打开浏览器开发者工具（F12），查看 Network 标签页，确认 API 请求正常

## 常见问题

### Q: 后端启动失败，提示 "Access denied for user"

A: 检查 `E:\Codes\call-go\config\database.go` 中的数据库密码是否正确，当前密码是：`Qw13101192533`

### Q: 前端无法连接后端，提示 Network Error

A: 
1. 确认后端已启动：访问 `http://localhost:8080/health`
2. 检查 `E:\Codes\call\src\services\api.ts` 中的 `BASE_URL` 是否正确
3. 检查浏览器是否有 CORS 错误

### Q: 数据库表不存在

A: 重新执行数据库初始化脚本：`E:\Codes\call-go\database\create_db.sql`

### Q: go 命令找不到

A: 
1. 重启终端
2. 或者在当前终端运行：`$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")`
